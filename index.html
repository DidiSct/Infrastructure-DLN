Latar Belakang & Hasil Audit Lapangan (Cabang Cibitung)

Berdasarkan kunjungan kerja dan pemeriksaan komprehensif (checklist assessment) yang dilakukan pada infrastruktur IT di Kantor Cabang Cibitung yang saat ini berfungsi sebagai pusat server, ditemukan beberapa kondisi kritikal. Kondisi eksisting ini memicu inefisiensi operasional yang tinggi, risiko keamanan data yang masif, serta kendala stabilitas performa sistem pada cabang-cabang lainnya.

No Temuan Lapangan (Kondisi Eksisting) Tingkat Risiko Analisis Dampak Operasional & Keamanan
1 Database ditempatkan pada PC Desktop biasa (Windows Client), bukan pada perangkat Server dedicated. KRITIKAL Komponen hardware PC desktop tidak didesain untuk beban kerja 24/7. Tanpa fitur redundansi (seperti redundant PSU atau RAID storage), kerusakan hardware seketika dapat menyebabkan total downtime dan potensi kehilangan data permanen.
2 Kantor cabang lain mengakses aplikasi terintegrasi menggunakan software remote control AnyDesk. KRITIKAL Metode ini memakan bandwidth internet yang besar, tidak efisien, dan membatasi mobilitas staf cabang. Celah keamanan sangat terbuka lebar karena mengandalkan pihak ketiga tanpa enkripsi jaringan korporasi mandiri.
3 Mesin Database masih menggunakan software Microsoft SQL Server versi 2000. KRITIKAL Versi ini telah lama berstatus End of Life (EOL). Tidak ada lagi patch keamanan resmi, menjadikannya target empuk serangan Ransomware/Malware, serta tidak optimal pada arsitektur hardware modern.
4 Koneksi ke database dari cabang lain kurang stabil. TINGGI Mengakibatkan interupsi transaksi harian, proses input data sering mengalami timeout, serta menurunkan produktivitas akibat sering terputusnya koneksi AnyDesk secara mendadak.
2

Rekomendasi Target Arsitektur Jaringan (Sentralisasi Head Office)

Untuk mengatasi seluruh kendala fatal di atas, direkomendasikan sebuah pembaruan menyeluruh melalui skema Sentralisasi Pusat Data ke Head Office (HO) Jakarta dengan penerapan topologi EoIP (Ethernet over IP) + IPsec VPN Tunnel.

Langkah strategis ini akan mengeliminasi ketergantungan pada AnyDesk, meningkatkan engine database ke level enterprise, serta menjamin jalur komunikasi data yang terenkripsi aman dan stabil antar seluruh cabang PT DUTA LINTAS NUSA.

Akses Aplikasi Langsung

Seluruh kantor cabang dapat mengeksekusi aplikasi internal secara native dari komputer lokal masing-masing, langsung mengarah ke IP Server Pusat melalui terowongan VPN aman.

Virtualisasi Infrastruktur

Perangkat Dell Server T40 di Head Office dapat dimanfaatkan secara optimal menggunakan arsitektur virtualisasi (Hyper-V) untuk memisahkan database fisik utama dan virtual server hosting secara rapi.

Keunggulan Utama

Terowongan EoIP & IPsec (AES-256)

Jaringan multi-cabang akan dikonsolidasikan ke satu server database di Head Office Jakarta dengan keamanan super ketat. Tidak ada celah data langsung terekspos ke internet publik tanpa enkripsi.

4

Catatan Kritis & Rekomendasi Tambahan

Legalitas Lisensi Software

Pembelian Windows Server melalui e-commerce harus dipastikan mendapatkan tipe OEM, COA Resmi, atau Volume Licensing (VL) yang valid. Hindari lisensi ilegal berbiaya sangat murah demi aspek kepatuhan audit (compliance) dan kestabilan jangka panjang.

Proteksi Daya Listrik (UPS)

Server pusat di Jakarta wajib dilengkapi Uninterruptible Power Supply (UPS) dengan kapasitas minimal 1200VA - 2000VA yang memiliki fitur manajemen shutdown otomatis untuk mencegah kerusakan database saat kegagalan daya.

Strategi Backup Otomatis

Mengingat data terpusat di Head Office, wajib dikonfigurasi jadwal backup otomatis harian (incremental) dan mingguan (full) dari SQL Server 2019 menuju eksternal storage atau Cloud backup terisolasi.

Kestabilan Jalur Internet Cabang

Meskipun cabang menggunakan IP Publik Dinamis, pastikan kualitas latency dari ISP lokal masing-masing kantor cabang tetap optimal mengarah ke Head Office agar performa aplikasi terasa responsif.

5

Kesimpulan Laporan

Kondisi infrastruktur saat ini yang terpusat pada PC Client di Cabang Cibitung sudah tidak memadai untuk mendukung ekspansi operasional multi-cabang PT DUTA LINTAS NUSA. Penerapan migrasi fisik dan jaringan menuju Head Office Jakarta menggunakan topologi aman EoIP + IPsec merupakan langkah wajib guna melindungi integritas aset data, meningkatkan efisiensi kerja staf, serta membangun pondasi IT yang scalable.
